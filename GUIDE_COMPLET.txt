GUIDE COMPLET - Cook Note
=========================

OBJECTIF
--------

Cook Note est un carnet culinaire web pour organiser un vrai repertoire de
recettes sans perdre les variantes :

- fiches parents : Coulis, Chantilly, Cremes, Pates, Cookies, etc. ;
- selection d'une variante dans la fiche parent ;
- panier de courses combine entre plusieurs recettes ;
- mode cuisine avec checklist, progression et minuteurs ;
- fiches techniques pour les familles de recettes ;
- back-office protege pour maintenir le catalogue.

ARCHITECTURE
------------

- index.html, app.js, style.css : site public React sans build.
- recipes.js : source publique des recettes et fiches parents.
- server.js : serveur Node et API admin.
- admin-login.html, admin.html, admin.js, admin.css : back-office.
- assets/recipe-images/ : images de recettes livrees avec le site.
- assets/uploads/ : images importees depuis l'admin.
- scripts/validate-recipes.js : validation du catalogue.
- scripts/validate-quantities.js : validation des multiplicateurs et rendements.
- scripts/validate-ui.js : validation des garde-fous d'interface.
- scripts/validate-project-rules.js : verification que les regles Cook Note restent branchees.
- scripts/validate-production.js : verification sitemap, robots, service worker et anciens ids.
- scripts/audit-recipes.js : rapport de completude et de rangement.
- A_LIRE_EN_PREMIER.md : garde-fou maitre a lire avant toute intervention.
- AGENTS.md : pointeur court pour les agents de code.
- COOK_NOTE_RULES.md : source de verite des conventions utilisateur.
- backups/ : copies horodatees de recipes.js avant ecriture admin.

FICHES PARENTS
--------------

Une fiche parent contient variants :

  variants: [
    { id: 'coulis_fraise', label: 'Coulis fraise' },
    { id: 'coulis_framboise', label: 'Coulis framboise' }
  ]

Chaque variante garde son contenu complet et pointe vers son parent :

  master: 'coulis_maitre'

Regle importante : une recette reelle ne doit pas etre supprimee pour ranger le
catalogue sans raison. On ajoute une fiche parent, puis on rattache les recettes
existantes. Exception explicite actuelle : les anciens coulis individuels ont ete
retires car ils existent deja comme variantes dans la fiche parent Coulis.

PANIER COURSES
--------------

Depuis une fiche recette :

1. Ouvrir la bonne variante.
2. Cliquer "Ajouter aux courses".
3. Recommencer avec une autre recette.
4. Ouvrir "Panier courses".

La liste regroupe les ingredients par recette, sans modifier le texte original.

BACK-OFFICE
-----------

Le back-office gere :

- champs principaux de recette ;
- fiche parent ;
- variantes d'une fiche parent ;
- fiche technique ;
- import d'image ;
- sauvegarde avec backup.

Mot de passe recommande :

   COOK_NOTE_ADMIN_PASSWORD

admin.local.json reste possible pour le developpement uniquement.

VALIDATION
----------

Commande :

   npm run check

Elle verifie :

- syntaxe JS ;
- existence de chaque fiche parent ;
- existence et rattachement de chaque variante ;
- absence de recette orpheline ;
- liens internes data-goto ;
- presence des images locales.
- images uniques entre recettes feuilles ;
- regles de donnees : arrondis, vanille, cassonade/vergeoise, poivre ;
- service worker, sitemap et robots ;
- presence des garde-fous Cook Note.

Audit non bloquant :

   npm run audit:recipes

Rapports :

   reports/recipe-audit.md
   reports/recipe-audit.json

DEPLOIEMENT
-----------

Le site public peut etre servi comme fichiers statiques si l'admin n'est pas
necessaire sur l'hebergement public.

Si l'admin doit etre accessible en ligne, deployer le serveur Node avec :

   npm start

et definir obligatoirement :

   COOK_NOTE_ADMIN_PASSWORD

Avant mise en ligne :

1. npm run check
2. verifier l'accueil desktop/mobile ;
3. verifier une fiche parent et plusieurs variantes ;
4. tester le panier courses avec au moins deux recettes ;
5. tester /admin avec connexion ;
6. verifier qu'aucun fichier de documentation ne presente le site comme un outil local.
